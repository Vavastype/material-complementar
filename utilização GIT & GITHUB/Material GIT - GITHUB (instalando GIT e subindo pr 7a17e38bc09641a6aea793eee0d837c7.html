Configuração inicial GIT

Caso desejar a documentação para utilização do GIT esta neste site, mesmo assim continuaremos a configuração por aqui…
Observação: Primeiro termos que possuir a conta do GitHub e junto com usuário de utilização, caso não possuir, volta para o PDF (Material GITHUB (Criando conta GitHub & Iniciando Repositório)) para iniciar criação de sua conta padrão.
Agora que você tem o Git em seu sistema, você deve fazer algumas coisas para personalizar o ambiente Git. Você fará isso apenas uma vez por computador e o efeito se manterá após atualizações. Você também pode mudá-las em qualquer momento rodando esses comandos novamente.

O Git vem com uma ferramenta chamada git config que permite ver e atribuir variáveis de configuração que controlam todos os aspectos de como o Git aparece e opera.

Sua identidade - Carimbo

A primeira coisa que você deve fazer ao instalar Git é configurar seu Nome de Usuário e endereço de E-Mail. Isto é importante porque cada commit usa esta informação, e ela é carimbada de forma imutável nos Commits que você começa a criar:
git config --global user.name "Fulano de Tal"

git config --global user.email fulanodetal@exemplo.br
Esse é um exemplo de configuração para sua conta GIT
Caso depois do processo desejar conferir as configuração digite esse comando em seu GIT git bash, para verificar informação de sua conta.

Subindo projeto para o GitHub usando Git (Hospedagem).

git init //-> Criar novo repositorio
git add * //-> Adicionar o coteúdo dentro do repositorio GIT
git remote add origin "Nesse campo sem aspas cole o link do repositorio GitHub"
git branch -m main //-> Adicione um ponteiro para localização de remificação (commints)
git commint -m "Seu primeiro commint"
git push -u origin main //-> subindo o projeto para GitHub
Copiando o link usaremos para iniciar o Git em nosso material e subi-lo para GitHub…
Logo depois de fazer um push sobre o projeto, vai pedir autorização do GitHub aceitando finalizara a hospedagem do projeto.
Atualizando a pagina do repositório ficara seu material no GitHub.
